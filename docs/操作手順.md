# 競馬予想ツール 操作手順

## 「サーバー」とは？

このツールでは **「サーバー」= あなたのPCで動かすプログラム（Node.js）** のことを指します。

- ブラウザで http://localhost:3000 を開くと表示される画面を**用意している**
- Cookie と URL を受け取って、競馬クラスターにアクセスし、結果を返す**API**を提供している

**サーバーを起動する** = そのプログラムを実行して、いつでもブラウザから使える状態にすることです。  
**サーバーを再起動する** = 一度そのプログラムを止めて、もう一度実行し直すことです。

---

## 手順一覧

1. フォルダを開く  
2. サーバーを起動する（初回は npm install と playwright のインストール）  
3. ブラウザでツールを開く  
4. Cookie と URL を準備する  
5. 取得を実行する  
6. トラブル時の「サーバーを再起動する」方法  

---

## 1. フォルダを開く

1. **エクスプローラー**で、このツールのフォルダを開く。  
   例: `C:\Users\jochi\keiba-cursor`
2. フォルダの中が表示されている状態にする（`package.json` や `server.js` が見えていればOK）。

---

## 2. サーバーを起動する

### 初回だけ（まだやっていない場合）

1. **ターミナル（コマンドを入力する画面）を開く**
   - 方法A: フォルダを開いた状態で、エクスプローラーの**アドレス欄**をクリックして `cmd` と入力し Enter。
   - 方法B: Cursor や VS Code を使っている場合は、メニュー「ターミナル」→「新しいターミナル」を開き、表示されたターミナルで **このプロジェクトのフォルダにいる**ことを確認する（画面に `keiba-cursor` などと出ていればOK）。
2. 次のコマンドを**1つずつ**入力して Enter する。
   ```bash
   npm install
   ```
   終わったら次。
   ```bash
   npx playwright install chromium
   ```
   （「ブラウザで取得」を使う場合だけ必要。時間がかかることがあります。）

### サーバーを起動する（毎回）

1. ターミナルで、**このツールのフォルダにいる**状態にする。  
   別のフォルダにいる場合は、次のように移動する。
   ```bash
   cd C:\Users\jochi\keiba-cursor
   ```
   （パスはあなたの環境に合わせて変えてよい。）
2. 次のコマンドを入力して Enter する。
   ```bash
   npm start
   ```
3. 画面に **「競馬予想ツール http://localhost:3000」** のような表示が出れば、サーバーが起動している。  
   **このターミナルは閉じないでおく**（閉じるとサーバーが止まります）。

---

## 3. ブラウザでツールを開く

1. **Chrome や Edge などのブラウザ**を開く。
2. アドレス欄に次のURLを入力して Enter する。  
   **http://localhost:3000**
3. **トップページ**で目的を選ぶ。  
   - **今日のレースを追う** → 当日のデータ取得・判定（`/today.html`）  
   - **分析・データ蓄積** → 過去用 Cookie の保存と手順（`/analyze.html`）

---

## 4. Cookie と URL を準備する

### URL の準備

1. **別のタブ**で [https://web.keibacluster.com/top](https://web.keibacluster.com/top) を開く。
2. **中央競馬**をクリックし、その日の**パスワード**を入力して**認証**する。
3. **会場**（例：中山）を選ぶ。
4. **レース一覧**で、見たいレース（例：1R）を**右クリック**して**「新しいタブで開く」**を選ぶ（またはレースリンクを**中ボタン・ホイールクリック**で開く）。
5. 開いたタブの**アドレス欄のURL**をコピーする。  
   **`?race_id=数字`** が含まれていること（例: `https://web.keibacluster.com/top/race-analyze?race_id=361645`）を確認してください。  
   ※ 通常クリックだとアドレスが変わらないことがあるため、「新しいタブで開く」で取得する必要があります。

### Cookie の準備

1. 競馬クラスターの**出馬表が表示されているタブ**のまま、**F12** キーを押して開発者ツールを開く。
2. 上にある **「アプリケーション」**（または Application）タブをクリックする。
3. 左側の **「Cookie」** の横の **▶** をクリックして開く。
4. **「https://web.keibacluster.com」** をクリックする。
5. 右側に Cookie の一覧が出る。  
   **laravel_session** の行をクリックし、下の「Cookie の値」を**すべて選択してコピー**する。
6. 同様に **XSRF-TOKEN** の行をクリックし、その値もコピーする。
7. ツールの Cookie 欄には、次のどちらかで貼り付けてよい。  
   - **2行で貼る**: 1行目に laravel_session の値、2行目に XSRF-TOKEN の値（ツールが自動で補完する）。  
   - **1行で貼る**: `laravel_session=値; XSRF-TOKEN=値` の形。値だけの1行でも、ツールが `laravel_session=` を付けて解釈する。

### Cookie の用途分け（今日用と過去・DB用）

ツールでは **2種類のセッション**をサーバーに別々に保存できます。

| 用途 | いつ使うか | 保存・環境変数 |
|------|------------|----------------|
| **今日・リアルタイム用** | 当日のレースを追う・データ取得 | 「サーバーに保存」で *今日用* を選ぶ / `KEIBA_COOKIE` |
| **過去・DB用** | 過去の開催日の画面を見る・将来のDB取り込み用 | 「サーバーに保存」で *過去・DB用* を選ぶ / `KEIBA_COOKIE_ARCHIVE` |

取得するときは **「保存済みのどちらを使うか」** のラジオで、上記のどちらを参照するか切り替えます。Cookie 欄に**直接貼った文字列**は、どちらのラジオでもその内容が優先されます。

## 5. 取得を実行する

### 5-A. データ取得・更新（日付・会場を選ばない・おすすめ）

レース一覧ページに載っている **全会場**について、**今の時刻（サーバー側では JST）に最も近い発走**のレースを自動で1件ずつ選び、**買い／見送り・買い目・オッズ表**までまとめて表示します。

1. 手順4のとおり **Cookie** を用意する（**レース一覧を一度開けるログイン状態**であればOK。特定レースのURLは不要）。
2. iPhone の場合は **「保存済みの Cookie を使う」** にチェックでも可。
3. **「データ取得」** をクリックする。  
   - 会場の数だけ出馬表を取りに行くため、**30秒〜1分半**かかることがあります。  
4. 画面に **会場ごとのカード**（RPT・判定・小さな出馬表）が並べば成功。  
5. **「更新」** を押すと、**その時点のオッズ**で同じ処理をやり直します。時刻が進んで **選ばれるレースが変わる**こともあります。

※ 開催日は **JST の「今日」** を優先し、データに無ければ **直近の開催日** を使います。  
※ うまく動かないときは、折りたたみ **「従来どおり：日付・会場を選んでレース一覧」** から従来の手順で取得できます。  
※ **Render** 等で **HTTP リクエストが30秒程度で打ち切られる**プランのとき、会場数が多いとタイムアウトすることがあります。ダッシュボードの **Request timeout** を **120秒以上** に伸ばすか、従来手順で会場を分けて取得してください。

### 5-B. URL を指定して1レースだけ取得する

1. ツールのタブ（http://localhost:3000）に戻る。
2. **「Cookie」**の欄に、手順4で作った **Cookie の1行**を貼り付ける。
3. **「データページのURL」**の欄に、手順4でコピーした **URL** を貼り付ける。
4. **通常の取得**でトップページが返る場合は、**「ブラウザで取得」**にチェックを入れる。
5. **「取得」**ボタンをクリックする。
6. 十数秒待つ（「ブラウザで取得」のときは特に時間がかかることがある）。
7. 結果が表示される。  
   - RPT と馬番・BB指数・単勝オッズの表が出れば成功。  
   - 「パースできませんでした」やヒントが出た場合は、表示されている説明に従って Cookie や URL を確認する。

---

## 6. トラブル時：サーバーを再起動する

「通信エラー」や「Unexpected token」が出たとき、またはコードを変更したあとで反映させたいときに使います。

1. **サーバーを動かしているターミナル**を開く（手順2で `npm start` を実行したあの画面）。
2. そのターミナルで **Ctrl + C** を押す。  
   → サーバーが止まり、「競馬予想ツール http://localhost:3000」の表示が消える。
3. もう一度、次のコマンドを入力して Enter する。  
   ```bash
   npm start
   ```
4. また「競馬予想ツール http://localhost:3000」と表示されれば、再起動完了。
5. ブラウザで http://localhost:3000 を**再読み込み（F5）**して、もう一度「取得」を試す。

---

## 7. 「address already in use :::3000」と出るとき

**意味** … ポート 3000 がすでに別のプログラム（前のサーバーなど）に使われている。

### 対処A：前のサーバーを止める

1. **別のターミナル**を開いている場合、そこで `npm start` していたなら、そのターミナルを開いて **Ctrl + C** で止める。
2. それでも解消しない場合は、下の「対処B」でポート 3000 を使っているプログラムを終了する。

### 対処B：ポート 3000 を使っているプログラムを終了する（Windows）

1. **新しいターミナル**（または「コマンドプロンプト」）を開く。
2. 次のコマンドを入力して Enter する。  
   ```bash
   netstat -ano | findstr :3000
   ```
3. 表示された行の**いちばん右の数字**（PID）をメモする。例: `12345`
4. 次のコマンドを入力して Enter する（`12345` の部分を、メモしたPIDに置き換える）。  
   ```bash
   taskkill /PID 12345 /F
   ```
5. もう一度、ツールのフォルダで `npm start` を実行する。

## 検証スクリプト（Cookie と URL の確認）

ツールの画面を使わず、**Cookie と URL が正しく動くか**だけを手元で確認したいときに使います。

1. **scripts/verify-input.txt** を開く。
2. **1行目**にデータページのURL（`?race_id=数字` 付き）を書く。
3. **2行目以降**に Cookie を貼り付ける（1行で `値1;値2` でも、2行で laravel_session と XSRF-TOKEN の値でも可）。
4. ターミナルで次を実行する。  
   ```bash
   npm run verify
   ```
5. 結果が **✅ 出馬表を取得できました** なら、同じ Cookie と URL でツールの「取得」も動く可能性が高いです。**❌ トップページが返っています** なら、Cookie の取り直しや有効期限を確認してください。

## まとめ（毎回やること）

| 順番 | やること |
|------|----------|
| 1 | ターミナルで `cd C:\Users\jochi\keiba-cursor` でフォルダに移動 |
| 2 | `npm start` でサーバー起動（ターミナルは閉じない） |
| 3 | ブラウザで http://localhost:3000 を開き、トップから「今日のレース」または「分析・蓄積」を選ぶ |
| 4 | 競馬クラスターでログイン→**Cookie をコピー**（まとめ取得なら一覧が見える状態でOK） |
| 5 | **「データ取得」**で全会場まとめて判定、または Cookie＋URL で下の「取得」 |

問題が出たら、手順6のとおり**ターミナルで Ctrl+C → もう一度 npm start** でサーバーを再起動してから、再度お試しください。
