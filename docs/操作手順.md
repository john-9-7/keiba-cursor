# 競馬予想ツール 操作手順（本番サーバー向け）

このドキュメントは、**インターネット上の本番サーバー**にデプロイしたツールを使う手順です。  
手元のPCで `npm start` する手順は **付録A** にあります。

> **蓄積データ（スナップショット・結果・確認画面）について**  
> **基本的に手元PCで `npm start` し、`http://localhost:3000` の「分析・データ蓄積」「蓄積の確認・結果」だけを使う運用を前提にしています。** データはPCの **`data/accumulated/`** に残ります。  
> クラウドのURLだけで蓄積すると、無料プランでは再デプロイで消えることがあります。詳細は **§7**。

---

## 「本番サーバー」とは？

- **本番URL** … ブラウザでアクセスするアドレスです（例: `https://あなたのアプリ名.onrender.com`）。  
  **Render・Railway・自前VPS** など、デプロイ先ごとに異なります。**デプロイ時に表示されたURLをメモし、ブックマーク**してください。
- **ツールのサーバー** … クラウド上で動いている **Node.js のプログラム**です。あなたが毎回起動する必要はありません（ホスティング側で常時起動、またはアクセス時に起動）。
- **API** … ブラウザから Cookie や race_id を送ると、競馬クラスターに取りに行き、結果を返す仕組みです。

> **この手順書内の `https://本番のURL`** は、必ず **ご自身の本番アドレス** に読み替えてください。

---

## 手順一覧（本番）

1. ブラウザで本番のツールを開く  
2. （推奨）ホスティングの環境変数で Cookie を登録する  
3. 競馬クラスター側で Cookie・URL を準備する  
4. 画面上で Cookie を保存する（または環境変数だけ使う）  
5. データ取得・分析を実行する  
6. トラブル時（本番）  
7. 蓄積データの場所（手元PCで運用）

## 1. ブラウザで本番のツールを開く

1. **Chrome や Edge などのブラウザ**を開く。
2. アドレス欄に **本番URL** を入力して Enter する。  
   例: `https://本番のURL`  
3. **トップページ**が表示されたら成功です。目的に応じて次を選びます。  
   - **今日のレースを追う** → `/today.html`（当日のデータ取得・判定。本番URLでも可）  
   - **分析・データ蓄積 / 蓄積の確認** … **手元PC** で `npm start` したうえでの利用を推奨（下記 §7）

### 会場12レース分をファイルに溜める（分析用・手元PC推奨）

1. 手元PCでプロジェクトを開き、`npm start` する。ブラウザで **http://localhost:3000/analyze.html** を開く。
2. **分析・データ蓄積**で過去・DB用 Cookie を保存する（手順4・付録Aと同様）。  
3. **「② レース一覧を取得する」** で開催リストを取り込み、プルダウンで選ぶ（または日付・会場を手入力）。
4. **「④ この日付・会場の全レースを保存する」** を押す。  
   **または** **「全会場まとめて蓄積する」** で一気に取り込む（時間がかかります）。
5. データは **そのPC上** の **`data/accumulated/snapshots.jsonl`** に追記されます。バックアップは **`data/accumulated/` フォルダごとコピー**で十分です。詳細は `docs/ACCUMULATOR.md`。

※ クラウド本番のURLだけで同じ操作をすると、ホスティングにより**再デプロイでファイルが消える**ことがあります（蓄積はPC運用を推奨します）。

### 蓄積の確認・着順の入力・簡易分析（手元PC）

1. 手元PCで **http://localhost:3000/accumulate-view.html** を開く。
2. **一覧**で `race_id`・RPT・判定・結果の突合、**1〜3着のBB順位**を確認する。**CSVダウンロード**で Excel に落とせます。
3. レース後、**結果を登録**フォームに **race_id** と **1〜3着馬番**（任意で着順まとめ）を入れ、**結果を保存**を押す。
4. **集計**が更新されます（直近最大500件を対象。RPT別の件数や「買い判定かつBB最高馬が1着」などの参考指標）。

## 2. （推奨）ホスティングの環境変数で Cookie を登録する

**iPhone だけで使う**・**画面に長い Cookie を貼りたくない**場合に便利です。管理画面は **Render / Railway / その他** で画面が異なりますが、考え方は同じです。

### 今日・リアルタイム用

| 変数名 | 内容 |
|--------|------|
| `KEIBA_COOKIE` | 競馬クラスターの Cookie 文字列（`laravel_session=...; XSRF-TOKEN=...` 形式に近いもの） |

### 過去開催・DB取り込み用

| 変数名 | 内容 |
|--------|------|
| `KEIBA_COOKIE_ARCHIVE` | 過去の開催日を見られる状態の Cookie |

### 登録後の作業

- 環境変数を変更したら、ホスティングの **「再デプロイ」** または **「Restart service」** でアプリを再起動し、反映を待ちます（サービスにより名称が異なります）。

### （任意）Basic 認証をかける場合

本番を公開する際、知られたURLにアクセスされないよう **環境変数** で Basic 認証を有効にできます。

| 変数名 | 例 |
|--------|-----|
| `ENABLE_AUTH` | `true` |
| `AUTH_USER` | ログイン用ユーザー名 |
| `AUTH_PASS` | ログイン用パスワード |

設定後、ブラウザで本番URLを開くと **ID・パスワードの入力**が求められます。

## 3. 競馬クラスター側で Cookie・URL を準備する

### URL の準備（1レースずつ取りたいとき）

1. **別のタブ**で [https://web.keibacluster.com/top](https://web.keibacluster.com/top) を開く。
2. **中央競馬**をクリックし、その日の**パスワード**を入力して**認証**する。
3. **会場**（例：中山）を選ぶ。
4. **レース一覧**で、見たいレース（例：1R）を**右クリック**して**「新しいタブで開く」**を選ぶ（またはレースリンクを**中ボタン・ホイールクリック**で開く）。
5. 開いたタブの**アドレス欄のURL**をコピーする。  
   **`?race_id=数字`** が含まれていること（例: `https://web.keibacluster.com/top/race-analyze?race_id=361645`）を確認してください。

### Cookie の準備

1. 競馬クラスターの**出馬表が表示されているタブ**のまま、**F12** キーで開発者ツールを開く。
2. **「アプリケーション」**（Application）タブ → 左の **Cookie** → **https://web.keibacluster.com**。
3. **laravel_session** の「値」をすべてコピーする。
4. **XSRF-TOKEN** の値もコピーする。
5. ツールの Cookie 欄には次のどちらかで貼ってよい。  
   - **2行**: 1行目＝laravel_session の値、2行目＝XSRF-TOKEN の値  
   - **1行**: `laravel_session=値; XSRF-TOKEN=値` ／ 値だけでもツールが解釈します。

## 4. 画面上で Cookie を保存する（環境変数の補助）

環境変数に `KEIBA_COOKIE` 等を入れていれば、**この手順はスキップ**して、画面上で **「保存済みの Cookie を使う」** だけでも動きます。

1. 本番の **今日のレース** または **分析・データ蓄積** を開く。
2. Cookie 欄に手順3でコピーした内容を貼る。
3. **今日用**／**過去用** は画面の説明に従う（当日だけなら通常は今日用で足ります）。
4. **サーバーに保存** ボタンを押す。  
   - 本番では、保存先は **サーバー（コンテナ）内のファイル** です。無料ホスティングでは **再デプロイで消える** ことがあります。**長期運用は手順2の環境変数**を推奨します。

---

## 5. 取得を実行する

### 5-A. データ取得・更新（全会場まとめて・おすすめ）

1. **Cookie** が使える状態にする（手順2の環境変数、または手順4の保存、または Cookie 欄に直接貼る）。
2. iPhone では **「保存済みの Cookie を使う」** にチェック（環境変数が効いていれば空欄でOK）。
3. **「データ取得」** をクリック。会場の数だけ取得するため **30秒〜1分半** かかることがあります。
4. 会場ごとのカードが並べば成功。**「更新」** でその時点のオッズに更新されます。

※ **Render** 等で **リクエストが約30秒で打ち切られる** プランでは、会場が多いとタイムアウトすることがあります。ダッシュボードの **Request timeout を 120秒以上** にするか、**従来手順**で会場を分けて取得してください。

### 5-B. URL を指定して1レースだけ取得する

1. 本番の **今日のレース** ページを開いたまま、下のエリアへスクロールする。
2. **Cookie** と **データページのURL**（`?race_id=` 付き）を入力する。
3. 必要なら **「ブラウザで取得」** にチェック（本番で Playwright が有効な場合のみ意味があります）。
4. **「取得」** をクリックし、結果を待つ。

## 6. トラブル時（本番）

### まず試すこと

1. ブラウザで **スーパーリロード**（Ctrl+F5）またはキャッシュ削除後に再読み込み。
2. 競馬クラスターで **もう一度ログイン**し、Cookie を取り直してから、環境変数または画面の保存を更新。
3. ホスティングの **ログ（Logs）** を開き、エラーメッセージがないか確認。

### アプリの再起動・再デプロイ

- **Render** 例: ダッシュボード → 該当 Web Service → **Manual Deploy** → **Deploy latest commit**、または **Restart**。
- コードや環境変数を変えたあとは、**必ず再デプロイまたは再起動**してからブラウザで再度試してください。

### ローカル開発時の「npm 再起動」について

手元のPCで `npm start` している場合だけ、ターミナルで **Ctrl+C** → もう一度 **`npm start`** で再起動します。本番では通常不要です。

## 7. 蓄積データの場所（手元PCで運用）

| 場所（このPC内） | 内容 |
|------------------|------|
| `data/accumulated/snapshots.jsonl` | レーススナップショット |
| `data/accumulated/results.jsonl` | 着順メモ（任意） |

- 手元PCで `npm start` しているとき、蓄積は **プロジェクトフォルダ直下の `data/accumulated/`** にあります。再デプロイの影響は受けません。
- **バックアップ** … `data/accumulated` フォルダを別ドライブやUSBにコピーするだけで十分です。  
- 環境変数 **`KEIBA_DATA_DIR`** を付けた場合のみ、保存先のルートが変わります（上級者向け）。

### クラウド本番のURLで蓄積機能を触った場合

無料のクラウド環境では **一時ストレージ**のことが多く、**再デプロイで蓄積が消える**ことがあります。**蓄積の取り込み・確認・CSVは、基本PCの localhost で行う**のが安全です。

### スマホのブラウザから同じPC上のツールを開きたい場合（任意）

必須ではありません。必要なときだけ **[手元PCトンネル.md](./手元PCトンネル.md)** を参照してください。

### リポジトリの `render.yaml` について

本番Web用の Blueprint 例です。**蓄積データの保存先としては想定していません**（永続ディスクなしの Free 向け）。

## 付録A：手元のPCでサーバーを動かす（蓄積・分析の主運用）

蓄積を使うときは、**毎回「サーバー起動 → ブラウザで localhost」**です。以下は **Windows** を想定した詳細手順です。

### A-1. 初回だけ（Node.js とプロジェクトの準備）

1. **Node.js を入れる**  
   [https://nodejs.org/](https://nodejs.org/) から **LTS** をダウンロードしてインストールする。  
   インストール後、一度PCを再起動すると確実なことがあります。

2. **プロジェクトフォルダを開く**  
   例: `C:\Users\あなたの名前\keiba-cursor`（クローンした場所）。

3. **PowerShell をそのフォルダで開く**  
   エクスプローラーでフォルダを開き、住所欄に `powershell` と入力して Enter（またはフォルダ内で Shift+右クリック → 「PowerShell ウィンドウをここで開く」など）。

4. **依存パッケージのインストール（初回のみ）**

   ```powershell
   npm install
   ```

   エラーが出なければ完了です（数分かかることがあります）。

5. **（任意・初回のみ）ブラウザ自動取得を使う場合**  
   トップや取得画面で「ブラウザで取得」を使うなら、続けて:

   ```powershell
   npx playwright install chromium
   ```

   蓄積・分析の**最短実行だけ**なら、必須ではありません。

### A-2. 使うたびにやること（起動）

1. **PowerShell** で、もう一度プロジェクトフォルダに移動する。

   ```powershell
   cd C:\Users\jochi\keiba-cursor
   ```

   （パスは自分の環境に合わせて変更してください。）

2. **サーバーを起動する**

   ```powershell
   npm start
   ```

3. PowerShell に **「開くURL: http://localhost:3000」** のような表示が出て、**カーソルが止まったまま**になるのを確認する（ポート3000が埋まっているときは **3001** などになることがあります。そのときは表示されたURLを開く）。  
   **この PowerShell のウィンドウは閉じないでください。** 閉じるとツールが止まります。

4. **ブラウザを開く**（Chrome、Edge など）。

5. **アドレス欄**に、さきほど表示されたURLを入力する。多くの場合は次のとおりです。

   ```text
   http://localhost:3000
   ```

6. **トップページ**が表示されたら成功です。

### A-3. 蓄積を使うときに開くページ（目安）

| したいこと | 開くアドレス（localhost のあと） |
|------------|-----------------------------------|
| データをファイルに溜める | `http://localhost:3000/analyze.html` |
| 溜めたデータを一覧・集計・CSV | `http://localhost:3000/accumulate-view.html` |
| 当日のレース | `http://localhost:3000/today.html` |

※ トップからリンクを辿っても同じです。

### A-4. データが保存される場所

- フォルダ **`プロジェクト内の data\accumulated\`**
- 中身の **`snapshots.jsonl`** がスナップショット蓄積の本体です。

### A-5. 終了するとき

- **PowerShell** で `npm start` を動かしている画面をアクティブにし、**Ctrl + C** を押す。  
- 止まったらウィンドウを閉じてかまいません。

### A-6. うまくいかないとき

| 症状 | 試すこと |
|------|----------|
| `localhost:3000` が開けない | `npm start` が動いているか。PowerShell に表示された **「開くURL: …」** と同じアドレスを使う。`http://127.0.0.1:3000` でも試す。 |
| `address already in use :::3000` | すでに別のターミナルで `npm start` が動いている。**もう一方で Ctrl+C** してから再度 `npm start`。 |
| `npm` が認識されない | Node.js のインストールが済んでいるか。パソコンを再起動したうえで、新しい PowerShell でやり直す。 |

## 付録B：検証スクリプト（手元のPCのみ）

`npm run verify` は **ローカル**の `scripts/verify-input.txt` を読むため、**本番ブラウザ操作の代わりにはなりません**。手元で Cookie と URL の動作確認をするときに使います。

## まとめ（本番で毎回やること）

| 順番 | やること |
|------|----------|
| 1 | ブラウザで **本番URL** を開く |
| 2 | （未設定なら）ホスティングで `KEIBA_COOKIE` 等を設定するか、画面から Cookie を保存する |
| 3 | 競馬クラスターでログインし、必要なら Cookie を取り直す |
| 4 | **「データ取得」** で全会場まとめて判定、または Cookie＋URL で **「取得」** |

問題が出たら **手順6**（再読み込み・Cookie 取り直し・ホスティングのログ確認・再デプロイ／再起動）を試してください。
