# 直前オッズでの再判定

## 概要

RPT・BB指数は出馬表確定後**変わらない**ため、一度取得すれば当日中はキャッシュして再利用できます。  
**オッズだけ**発走直前で変動するため、「キャッシュの RPT・BB ＋ ユーザー入力の直前オッズ」で再判定します。

## 使い方

1. 先に **「データ取得」** または **「レース詳細の取得」** で RPT・BB を取得する（1回成功すればキャッシュされる）
2. **「直前オッズで再判定」** を開く
3. **race_id** を入力（カードのボタンから開いた場合は自動入力）
4. **馬番=オッズ** をカンマ区切りで入力。例: `4=3.2, 8=15.5, 1=4.1`
5. **再判定** を押す

## 入力フォーマット

- `馬番=単勝オッズ` をカンマ・スペース・全角カンマ・頓号で区切り
- 例: `4=3.2, 8=15.5, 1=4.1` または `4=3.2 8=15.5 1=4.1`
- 未入力の馬番はオッズ `null` として判定（オッズ足切り等のロジックに影響）

## キャッシュ

- **保存タイミング**: レースの取得に成功したとき（ダッシュボード・fetch-race）
- **リセット**: 翌日 0 時（JST）で自動失効
- **保存先**: `data/race-cache.json`（.gitignore 対象）

## API

```
POST /api/judge-with-odds
Content-Type: application/json

{ "raceId": 362023, "oddsText": "4=3.2, 8=15.5, 1=4.1" }
```

## 今後の拡張（OCR）

画像からオッズを読み取る OCR 対応を検討可能。その場合はテキスト入力に加えて「画像をアップロード」を追加する形になります。
