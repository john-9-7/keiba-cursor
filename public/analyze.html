最短手順（通常はこの順だけ）
  1. 競馬クラスターで過去の開催日の画面を開き、当日用とは別にログイン（必要ならその日のパスワード）します。
  2. 開発者ツール（F12）→ アプリケーション → Cookie から laravel_session と XSRF-TOKEN をコピーします。
  3. 下の欄に貼り付け、「最短実行」を押します（保存→蓄積→結果取得まで自動）。
  4. 以降は Cookie が期限切れになるまで、Cookie欄は空で再実行できます。

保存先の詳細（通常は見なくてOK）

ツールが動いているサーバー内のフォルダ data/accumulated/ に保存されます（本番URLで使っているときはクラウド上。手元PCで npm start のときはそのPCのプロジェクト内）。

  • snapshots.jsonl … 各レースの RPT・馬柱・判定など（1行＝1レース・追記のみ）
  • results.jsonl … 後から着順を足すとき用（API /api/accumulate/result）

詳細は docs/ACCUMULATOR.md。バックアップはフォルダごとコピーしてください（.jsonl は Git に含まれません）。

一覧の全会場・全レースを一気に取り込む（超一括）

レース一覧ページに その Cookie で表示されている すべての「日付×会場」について、各会場の全レースを順に蓄積します。手動で会場を選ぶ必要はありません。

例: 開催が1日だけなら空欄でOK。複数日が一覧に混ざるときは、1日分だけ取りたい日付を入れるとその日の会場だけ処理します。

注意：会場数×レース数だけAPIを叩くため数分〜十数分以上かかります。Render の無料枠など HTTP タイムアウトが短いと途中で切れることがあります。そのときは Render の timeout 延長、または下の「1会場ずつ（①〜④）」を利用してください。
重複：同じ race_id は既に蓄積済みなら追記しません（再実行しても二重になりません）。

会場の全レースを一括で蓄積（〜12R）
